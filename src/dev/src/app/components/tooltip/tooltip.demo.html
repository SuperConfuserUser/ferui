FerUI Tooltip Component

Overview

A tooltip is a user-triggered UI element that displays supplementary information. The FerUI tooltip is a directive that generates a tooltip message on mouse hover or focus events for the paired element. Tooltips may also be closed with the 'ESC' key.

Some UI guidelines for tooltips include:

Usage

Requires: Import FeruiModule to your app or ngModule.

Add a FerUI tooltip to HTML elements or custom components using the fuiTooltip selector to assign content. As an attribute directive, it must be used with another element. Optionally, customize the FerUi tooltip with the fuiTooltipConfig input. Hover over or tab through code styled text in this paragraph to see the tooltip in action. (Click text first to set general focus and make tabbing easier.)

Interfaces and Types

Details for various interfaces and type aliases you may come across when using the FerUi tooltip.

TooltipComponentTypeContent

Object used to configure component type content for the tooltip.

Property Type Description
component Type<any> [Mandatory] The component to generate for the tooltip content. This would simply be the name of your component.
input {{ '{ [key: string]: any }' }} [Optional] Any additional input values for this component. This object will pass values by key (input name) to the component instance.
Component type content is dynamically compiled, so don't forget to add the component to your ngModule entryComponents section.

FuiTooltipConfig

An optional configuration object for customizing the tooltip. The configured properties will override default values.

Property Type Description Default Value
offset number [Optional] The distance that the tooltip will be placed from its parent element in pixels. {{ TOOLTIP_DEFAULT_CONFIG.offset }}
placement FuiTooltipPlacementEnum [Optional] The placement of the tooltip relative to its parent element. The tooltip will reposition to another placement if it does not fit the current view. FuiTooltipPlacementEnum.TOP
cssClass FuiTooltipThemeEnum | string [Optional] The pre-built theme classes or any custom CSS classes to apply to the tooltip. Multiple class names are allowed and should be space separated in the string. FuiTooltipThemeEnum.INFO
tooltipStyle string [Optional] Any additional styling to apply to the tooltip. Use the HTML/CSS inline style format. null
arrow boolean | string [Optional] A boolean value determines whether to show the default arrow, and a string value is for custom svg arrows. Be sure to also update arrowSize as needed for final scaling. Use SVG width, height and viewBox attributes for more consistency. {{ TOOLTIP_DEFAULT_CONFIG.arrow }}
arrowSize ArrowSize [Optional] The tooltip arrow size in pixels. {{ '{ width: '}}{{ TOOLTIP_DEFAULT_CONFIG.arrowSize.width }}{{', height: '}}{{ TOOLTIP_DEFAULT_CONFIG.arrowSize.height }}{{' }' }}
isDisabled boolean [Optional] Whether the tooltip should be disabled and not react to any events. {{ TOOLTIP_DEFAULT_CONFIG.isDisabled }}
fixedParent boolean [Optional] Whether the tooltip's parent element is position: fixed. This allows the tooltip to be positioned accurately. The special cases section includes an example. {{ TOOLTIP_DEFAULT_CONFIG.fixedParent }}

FuiTooltipPlacementEnum

Placement options for the tooltip relative to its paired element. The default is FuiTooltip.TOP.

FuiTooltipThemeEnum

Available pre-built theme classes that may be assigned to the tooltip. These are similar to the FerUi alert and button themes. The default is FuiFuiTooltipPlacementEnum.INFO.

ArrowSize

Object used to configure the tooltip arrow's size. The default is {{ '{ width: '}}{{ TOOLTIP_DEFAULT_CONFIG.arrowSize.width }}{{', height: '}}{{ TOOLTIP_DEFAULT_CONFIG.arrowSize.height }}{{' }' }}.

Property Type Description Default Value
width number [Mandatory] The width of the tooltip arrow in pixels. {{ TOOLTIP_DEFAULT_CONFIG.arrowSize.width }}
height number [Mandatory] The height of the tooltip arrow in pixels. {{ TOOLTIP_DEFAULT_CONFIG.arrowSize.height }}

Style

All FerUi tooltips have a base .fui-tooltip class. Then, the theme or custom class(es) is added. CSS classes are assigned in HTML format (e.g. 'fui-tooltip tooltip-info'). Some inline style is also generated to position the tooltip.

Any custom classes or styling assigned via FuiTooltipConfig will be applied according to typical CSS cascade rules (source order, specificity, importance). SVG style attributes like fill will be passed to the tooltip's SVG arrow.

The default z-index value is designed to work within the overall FerUI design framework along with other carefully established z-indexes. This value may need to be updated to work well within other existing systems and projects. Simply override the .fui-tooltip z-index attribute in a global stylesheet.
Angular styles are scoped to a single component by default. Make sure any custom CSS classes are available to the TooltipComponent. They should be included in a global stylesheet, exposed to the global style scope via encapsulation: ViewEncapsulation.None, or another method. Be careful to manage your global CSS classes well.

Special Cases

Disabled Elements

Some browsers (Chrome) may not allow events on disabled elements, so the tooltip won't trigger. Add the tooltip to a wrapper around the disabled element instead.

Fixed Positioned Parents

If the parent element of the tooltip directive is position: fixed, the tooltip will need to be configured for accurate positioning. Just set the fuiTooltipconfig.fixedParent property to true. In the demo below (visible when toggled), test tooltips with and without the fix by hovering on the buttons and scrolling.

Content Types

Tooltip content is assigned directly with the fuiTooltip selector. Content may be a string, HTML, template or component. Components used for tooltip content should be available to the FuiTooltipDirective as an entry component in your ngModule. Component type content may also receive input values as shown in the 'Advanced Component' button.

I'm a template!

Placement

Tooltip placement is assigned via fuiTooltipConfig.placement and will be placed on '{{ TOOLTIP_DEFAULT_CONFIG.placement }}' by default. The tooltip may be placed in the following positions relative to its paired element (from the FuiTooltipPlacementEnum enum):

  • TOP
  • BOTTOM
  • RIGHT
  • LEFT

Auto Repositioning

If the tooltip does not fit the current view with the default or assigned placement, it will be repositioned. Tooltips in the demo below (visible when toggled) are configured with placements that match their button label. The demo buttons are placed in extreme positions, so their tooltips will be automatically repositioned for visibility.

Themes

Pre-built themes may be assigned to the tooltip via fuiTooltipConfig.cssClass and has FuiTooltipThemeEnum.INFO styling by default. These are similar to other FerUi themes such as those for the alert and button components which include (from the FuiTooltipThemeEnum enum):

  • PRIMARY
  • SECONDARY
  • SUCCESS
  • DANGER
  • WARNING
  • INFO
  • LIGHT
  • DARK

Classes and Styling

Custom tooltip classes and styles may be assigned via fuiTooltipConfig.cssClass and fuiTooltipConfig.style. Multiple class names are allowed and should be space separated just like how multiple HTML classes are assigned. Also use the HTML/CSS inline style format for custom styles. SVG style attributes such as fill will be passed to the tooltip's SVG arrow.

Make sure any custom CSS classes are available to the TooltipComponent. They should be included in a global stylesheet, exposed to the global style scope via encapsulation: ViewEncapsulation.None, or another method. Be careful to manage your global CSS classes well. The ViewEncapsulation method is used here, since this is only for demonstration purposes.

Arrow Options

Tooltip arrows may be configured via fuiTooltipConfig.arrow and fuiTooltipConfig.arrowSize. By default, the arrow is displayed at {{ TOOLTIP_DEFAULT_CONFIG.arrowSize.height }} pixels high and {{ TOOLTIP_DEFAULT_CONFIG.arrowSize.width }} pixels wide. A boolean value determines whether to show the default arrow, and a string value will be used for custom svg arrows. When using a custom arrow, be sure to also update fuiTooltipConfig.arrowSize as needed for final scaling and use SVG width, height and viewBox attributes for even more consistency.

Tooltip arrows may be styled with SVG attributes in custom CSS classes (e.g. fancy-tooltip in the styling section), styles (e.g. myStyleConfig in the styling section) or inline (examples right below).

Enable or Disable

The tooltip can be enabled or disabled via fuiTooltipConfig.isDisabled. It is enabled by default.
