Conveying meaning to assistive technologies

Using color to add meaning only provides a visual indication, which will not be conveyed to users of assistive technologies – such as screen readers. Ensure that information denoted by the color is either obvious from the content itself (e.g. the visible text), or is included through alternative means, such as additional text hidden with the .sr-only class.

Alert Flex positioning

We've also added a .alert-flex class along with different wrappers classes to display an alert into three separated parts:

  • .fui-alert-icon-wrapper: This is the left wrapper that usually contains the icons.
  • .fui-alert-body-wrapper: This is the center wrapper that usually contains the message.
  • .fui-alert-close-wrapper: This is the right wrapper that usually contains the close button.

Angular Component

With FerUI, we have created a special Angular component to handle alerts more easily. It automatically creates a Flex alert and positions the content appropriately. The closing of an alert is directly handled by this component so the developer doesn't have to take care of it manually. Also, regarding the alert type (success, info, error, warning... etc) we've created a TypeScript interface for you to easily choose what kind of alerts that you want. FuiAlert API

Here is the list of all fui-alert attributes and what they do.

Property Type Description

[alertType] FuiAlertsTypes The type of alert you want to display. Refer to the enum interface bellow.
